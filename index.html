04. 업무 폴더 설계

작업 순서 파일은 다섯 줄부터 채우면 됩니다

01

목표

끝났을 때 무엇이 만들어져야 하는지 씁니다.

02

읽을 파일

Agent가 먼저 읽어야 할 폴더와 자료 위치를 지정합니다.

03

작업 순서

사람이 머릿속으로 하던 판단 순서를 번호로 풀어냅니다.

04

금지선

확인 안 된 결론, 과장 표현, 확정하면 안 되는 내용을 막습니다.

05

결과와 검수

최종 결과물과 사람이 볼 검수표를 함께 요청합니다.

작업 순서 파일 초안 목표: 원본 데이터를 읽고 요약 보고서로 바꾼다. 읽을 파일: 00_역할과금지선, 01_이번재료, 02_반복기준 순서: 읽기 → 분리 → 작성 → 교정 금지: 확인 안 된 결론 단정 금지 결과: 요약표 + 개선 제안 + 검수 질문
채용 비유로 이해하기처음부터 채용 업무 전부를 맡기지 않듯이, 작업 순서도 JD 기준으로 이력서 1차 정리하기처럼 범위가 좁을수록 잘 작동합니다.
이 장의 결론처음 작업 순서는 완벽한 문서가 아니라, 이 다섯 줄을 채우는 것부터 시작하면 됩니다.

퍼실 질문 예시 1. 이 업무는 언제 반복돼요? 2. 이번에만 쓰는 자료는 뭐예요? 3. 다음에도 계속 지켜야 할 기준은 뭐예요? 4. 결과물이 어떤 모양이면 끝인가요? 5. 사람이 마지막에 꼭 봐야 하는 위험한 부분은 뭐예요? 목표는 완성된 Agent가 아니라, 자료와 기준을 나눠 첫 실행까지 가는 것입니다.
