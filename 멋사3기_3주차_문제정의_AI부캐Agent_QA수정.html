산출물

무엇이 나오면 끝인가

사람이 바로 검수하거나 사용할 수 있는 형태로 끝냅니다.

재사용

다음번에도 쓸 수 있나

한 번의 답변이 아니라 다음 작업에도 남는 기준으로 만듭니다.

03. 콘텐츠 편집장

단순 QA가 아니라 우리만의 품질 기준을 남깁니다

CORE STANDARD 매번 같은 기준으로
콘텐츠를 검수하게 만든다 좋은 Agent는 결과물을 한 번 고치는 것이 아니라, 다음 결과물에도 적용될 검수 기준을 남깁니다.
사실성방문하지 않은 경험, 확인되지 않은 효과, 과장된 감정을 만들지 않는다.
분리업체 조건과 내 실제 후기를 섞어 사실처럼 쓰지 않는다.
말투기존 블로그의 문장 길이, 표현 강도, 문단 리듬을 유지한다.
채널블로그, 인스타, 첫댓글이 각각 다른 채널 문법을 따르게 한다.
발행성최종본은 사람이 바로 복사해 쓸 수 있는 발행 패키지로 끝낸다.

  • 실제 샘플 데이터 5~10행

  • 결과를 보고 고친 Rule / Skill 변경점

마무리

3주차 핵심 정리

1. 만들기 쉬워졌습니다

그래서 정보 자체보다 사람의 경험과 맥락을 전달하는 기준이 중요해졌습니다.

2. Agent는 온보딩입니다

역할, 읽을 자료, 판단 기준, 금지선을 문서로 남기는 일입니다.

3. 기준은 문서화합니다

Input, Reference, Rule, Skill, Output, Check를 분리해야 결과가 안정됩니다.

4. 첫 Skill부터 시작합니다

폴더를 만들고, 첫 Skill 파일 하나를 쓰고, 실행 후 기준을 고치는 것이 시작입니다.

Closing

일을 없애는 것이 아닙니다

나만의 반복 업무가 같은 기준과 순서로 실행되게 만드는 것입니다.
