Agent Concept

AI Agent는 네 단계로 문제를 해결합니다

사용자 요청자연어로 목표, 조건, 원하는 결과를 전달합니다.
→
AI Agent자료를 읽고 판단하고 도구를 실행하며 결과를 남깁니다.
01

인식

대화, 문서, 데이터베이스, 웹 검색, 소셜 미디어 등에서 필요한 정보를 수집하고 현재 상황을 파악합니다.

02

추론

수집한 정보를 기준과 목표에 맞춰 해석합니다. 필요하면 검색 보강 생성(RAG)처럼 외부 지식을 함께 활용합니다.

03

행동

정해진 권한 범위 안에서 파일 작성, 도구 실행, 시스템 연동, 사람에게 넘기기 같은 작업을 수행합니다.

04

학습

실행 결과와 피드백을 기록해 다음 작업의 기준을 업데이트하고, 반복될수록 더 안정적인 결과를 만듭니다.

강의에서의 해석오늘 만드는 폴더와 Skill은 Agent가 인식할 자료와 추론할 기준을 남기는 장치입니다.
동시에 행동할 절차와 학습할 기록을 쌓는 구조입니다.

Intro Framework

잘하는 사람의 Agent는 여섯 가지를 먼저 고정합니다

Input

무엇을 읽나

실제로 확인할 원본 자료입니다.

Reference

무엇을 따라가나

말투, 취향, 좋은 예시, 과거 결과물입니다.

Rule

어떻게 판단하나

금지사항, 우선순위, 검수 기준입니다.

Output

무엇으로 끝나나

바로 쓸 수 있는 최종 패키지입니다.

Skill

어떤 순서로 일하나

분석, 작성, 변환, 검수처럼 반복되는 작업 절차입니다.

Check

무엇으로 검수하나

사람이 결과물을 확인할 기준과 실패 조건입니다.

Agent Fit

모든 일을 Agent로 만들 필요는 없습니다

Agent로 만들기 좋은 업무

  • 반복해서 맡기는 일
  • 자료를 읽고 판단해야 하는 일
  • 매번 비슷한 수정 요청이 나오는 일
  • 결과물을 사람이 검수할 수 있는 일
  • 좋은 예시와 실패 예시가 쌓이는 일

아직 Agent로 만들기 애매한 업무

  • 한 번만 하고 끝나는 일
  • 판단 기준이 전혀 없는 일
  • 결과물을 검수하기 어려운 일
  • 최종 책임을 넘기면 안 되는 의사결정
  • 입력 자료가 계속 바뀌는데 기준은 없는 일
선정 기준
“자주 하고, 기준이 있고,
결과물을 확인할 수 있는 일”부터 Agent로 만듭니다.
Input vs Reference

이번 재료와 따라야 할 기준을 섞지 않습니다

Input

이번 작업에서만 쓰는 자료

  • 이번 업체 가이드라인
  • 이번 방문 후기 메모
  • 이번 고객사 요청사항
  • 이번 후보 장소와 일정 조건
Reference

앞으로도 반복해서 따라갈 기준

  • 내 기존 말투와 좋은 제목 예시
  • 이전 강의 톤과 실습 예시
  • 좋았던 코스와 실패했던 코스
  • 검수 체크리스트와 금지사항
강의 포인트
자료를 한 번에 다 던지는 것이 아니라,
이번 재료와 반복 기준을 분리해야 결과가 안정됩니다.
Skill Design

Skill은 긴 프롬프트가 아니라 작업 절차입니다

01 READ
읽기 입력 자료에서 이번 작업에 필요한 조건과 제약을 추출합니다.
02 JUDGE
판단하기 Reference와 Rule을 기준으로 무엇을 반영하고 버릴지 고릅니다.
03 MAKE
만들기 정해진 Output 형식에 맞춰 사람이 바로 검수할 수 있는 초안을 만듭니다.
04 CONVERT
변환하기 블로그에서 인스타, 제출물에서 피드백 카드처럼 용도별 결과물로 바꿉니다.
05 CHECK
검수하기 금지사항, 누락 조건, 출력 형식을 마지막으로 확인합니다.
Skill 작성 기준
좋은 Skill은 “멋진 문장”이 아니라 매번 같은 순서로 따라 할 수 있는 작은 워크플로입니다.

Agent 01 · Why

그냥 “블로그 글 써줘”라고 하면 생기는 문제

광고문처럼 변함

블로그 긴 글, 인스타 짧은 캡션, 첫댓글, 해시태그가 한 톤으로 섞여 나옵니다.

편집 판단 기준

Output

발행 패키지

블로그 원고, 제목 후보, 인스타 캡션, 첫댓글, 해시태그, 검수표로 끝냅니다.

Final Summary

3주차 핵심 정리

1. 인트로 인사이트

Agent가 막히는 이유는 대개 도구가 아니라 문제정의, 범위, 기준 부재에 있습니다.

2. 멘토 사례로 학습

잘하는 사람은 콘텐츠, 피드백, 일정 같은 반복 업무를 기준과 흐름으로 바꿉니다.

3. 기준은 문서화

Input, Reference, Rule, Skill, Output, Check를 분리해야 결과가 안정됩니다.

4. 첫 Skill부터 시작

폴더를 만들고, 첫 Skill 파일 하나를 쓰고, 실행 후 기준을 고치는 것이 시작입니다.
